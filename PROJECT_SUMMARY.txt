================================================================================
                    MERN STACK TODO APPLICATION
================================================================================

PROJECT OVERVIEW
--------------------------------------------------------------------------------
Full-stack task management application with role-based access control (RBAC),
enabling users to create, manage, and track todos with admin oversight
capabilities.

TECHNICAL STACK
--------------------------------------------------------------------------------
Backend:
  • Node.js + Express.js (REST API)
  • MongoDB + Mongoose (Database & ODM)
  • JWT (Authentication)
  • bcryptjs (Password Hashing)

Frontend:
  • React 19 + Vite (UI Framework & Build Tool)
  • React Router v6 (Client-side Routing)
  • Axios (HTTP Client)
  • Tailwind CSS (Styling)

ARCHITECTURE & FEATURES
--------------------------------------------------------------------------------
Authentication System:
  • User registration and login with JWT token-based authentication
  • Password encryption using bcryptjs
  • Protected routes with middleware authorization
  • Role-based access control (User/Admin roles)

Todo Management:
  • Full CRUD operations (Create, Read, Update, Delete)
  • Todo attributes: title, description, category, due date, completion status
  • User-specific todo isolation
  • Category classification (Urgent/Non-Urgent)

Admin Dashboard:
  • View all users and their todos
  • User role management (promote to admin)
  • System-wide todo overview
  • User activity monitoring

API ENDPOINTS
--------------------------------------------------------------------------------
Authentication:
  POST /api/auth/register    - User registration
  POST /api/auth/login       - User authentication

Todo Operations (Protected):
  GET    /api/todos          - Fetch user todos
  POST   /api/todos          - Create new todo
  GET    /api/todos/:id      - Get specific todo
  PUT    /api/todos/:id      - Update todo
  DELETE /api/todos/:id      - Delete todo

Admin Operations (Admin Only):
  GET   /api/admin/todos           - View all todos
  GET   /api/admin/users           - View all users
  PATCH /api/admin/users/:id/role  - Update user role

DATABASE MODELS
--------------------------------------------------------------------------------
User Schema:
  • username, email, password (hashed)
  • role (user/admin)
  • timestamps

Todo Schema:
  • title, description, dueDate, category
  • completed (boolean)
  • user reference (ObjectId)
  • timestamps

SECURITY IMPLEMENTATION
--------------------------------------------------------------------------------
  • JWT token-based stateless authentication
  • Password hashing with bcrypt (salt rounds: 10)
  • Protected API routes with auth middleware
  • Role-based authorization middleware
  • CORS enabled for cross-origin requests

PROJECT STRUCTURE
--------------------------------------------------------------------------------
Server/
  ├── config/          - Database configuration
  ├── controllers/     - Business logic (auth, todo, admin)
  ├── middleware/      - Auth & role verification
  ├── models/          - Mongoose schemas (User, Todo)
  ├── routes/          - API route definitions
  ├── dtos/            - Data transfer objects
  └── scripts/         - Utility scripts (admin creation)

Frontend/
  ├── src/
  │   ├── components/  - Reusable UI components
  │   ├── pages/       - Route pages (Login, Register, Dashboard, Admin)
  │   ├── contexts/    - React Context (AuthContext)
  │   └── services/    - API integration (axios)

DEPLOYMENT CONFIGURATION
--------------------------------------------------------------------------------
Environment Variables Required:
  • PORT (default: 5000)
  • MONGO_URI (MongoDB connection string)
  • JWT_SECRET (Token signing key)

Run Commands:
  Backend:  npm run dev (nodemon) | npm start (production)
  Frontend: npm run dev (development) | npm run build (production)
